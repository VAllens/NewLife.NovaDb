# NewLife.NovaDb 竞品分析

本文档对比 NovaDb 与主流数据库在关系型、时序、NoSQL、消息队列四大场景下的差异，用于产品定位参考。

---

## 1. 关系型数据库对比

| 特性 | SQLite | MySQL | PostgreSQL | SQL Server | **NovaDb** |
| :--- | :--- | :--- | :--- | :--- | :--- |
| **实现语言** | C | C/C++ | C | C/C++ | **C#** |
| **开源协议** | Public Domain | GPL v2 | PostgreSQL License | 商业 | **MIT** |
| **运行形态** | 嵌入（库） | 独立服务 | 独立服务 | 独立服务 | **嵌入（库）/ 服务（进程）** |
| **部署方式** | 单文件 | 安装+配置 | 安装+配置 | 安装+配置 | **文件夹即数据库，零配置** |
| **存储结构** | B+ 树 / 单文件 | InnoDB 页/段 | 页/段 (堆表) | 页/区 | **每表独立文件组（.nvd + .nvi）+ MMF** |
| **索引结构** | B+ 树 (磁盘) | B+ 树 (磁盘) | B+ 树 / GIN / GiST | B+ 树 / 列存储 | **SkipList (内存+MMF 冷热分离)** |
| **内存管理** | 固定缓存池 | Buffer Pool (LRU) | 共享内存 (Buffer) | Buffer Pool | **动态冷热分离 (热段内存+冷段 MMF)** |
| **事务隔离** | Serializable | RC / RR / Serializable | RC / RR / Serializable | RC / SI / Serializable | **Read Committed (MVCC)** |
| **并发模型** | 单写多读 (WAL) | 多写多读 (MVCC) | 多写多读 (MVCC) | 多写多读 (锁/MVCC) | **MVCC 多读多写（嵌入跨进程额外表级文件锁）** |
| **迁移/备份** | 拷贝文件 | mysqldump / 主从 | pg_dump / 流复制 | bak 文件 / AlwaysOn | **拷贝文件夹** |
| **扩展能力** | 纯关系型 | 纯关系型 | 关系型+JSON | 关系型+JSON+图 | **关系型 + 时序 + MQ + KV** |
| **运维复杂度** | 极低 | 中 | 中 | 中偏高 | **低（单文件夹/单进程）** |
| **跨平台** | ✅ | ✅ | ✅ | Windows 为主 | **✅ (.NET 跨平台)** |
| **适用规模** | 小型/嵌入 | 中大型 | 中大型 | 中大型 | **中大型/嵌入** |
| **生态集成** | 通用 ADO | 通用 ADO | 通用 ADO | 通用 ADO | **NewLife XCode 原生集成** |

| **SELECT / WHERE / ORDER BY** | ✅ | ✅ | ✅ | ✅ | **✅** | |

| **聚合函数 (COUNT/SUM/AVG/MIN/MAX)** | ✅ | ✅ | ✅ | ✅ | **✅** | |
| **JOIN (INNER/LEFT/RIGHT)** | ✅ | ✅ | ✅ | ✅ | **⚠️ 基础 (Nested Loop)** | 初期不支持 Hash Join（规划支持） |
| **子查询 (IN/EXISTS)** | ✅ | ✅ | ✅ | ✅ | **⚠️ 基础** | |
| **CTE (WITH 递归)** | ✅ | ✅ (8.0+) | ✅ | ✅ | **❌** | |
| **窗口函数 (ROW_NUMBER 等)** | ✅ (3.25+) | ✅ (8.0+) | ✅ | ✅ | **❌** | |
| **视图 (VIEW)** | ✅ | ✅ | ✅ | ✅ | **❌** | |

| **触发器** | ✅ | ✅ | ✅ | ✅ | **❌** | |
| **存储过程/函数** | ❌ | ✅ | ✅ | ✅ | **❌** | |
| **事务 COMMIT/ROLLBACK** | ✅ | ✅ | ✅ | ✅ | **✅ (MVCC)** | Read Committed |

## 3. 时序数据库对比

| :--- | :--- | :--- | :--- | :--- | :--- |
| **实现语言** | Go (v2) / Rust (v3) | C | C (PG 扩展) | Java | **C#** |
| **开源协议** | MIT (v2) / Apache 2.0 (v3) | AGPL v3 | Apache 2.0 (社区版) | Apache 2.0 | **MIT** |
| **部署方式** | 独立服务 | 独立服务/集群 | PG 扩展 | 独立服务 | **嵌入 / 独立服务** |
| **数据模型** | Tag + Field + Timestamp | 超级表 (STable) | 普通表 + 时间分区 | 表 + 时间列 | **时间分片表 (按天/小时)** |
| **写入模式** | Line Protocol (追加) | 追加写 | SQL INSERT (追加) | 追加写 | **追加写 (Append Only)** |
| **存储引擎** | TSM / IOx (列存储) | VNode 分片 | 基于 PG 堆表 + 分区 | 列存储 (零 GC) | **时间分片文件 + WAL + MMF** |
| **查询语言** | Flux / InfluxQL / SQL (v3) | 类 SQL (TQL) | 标准 SQL | 标准 SQL | **SQL (扩展语法)** |
| **降采样/聚合** | 连续查询 (Flux) | 内置 INTERVAL | TimescaleDB Toolkit | 内置采样 | **计划支持（优先级较低）** |
| **数据保留 (TTL)** | ✅ (Retention Policy) | ✅ | ✅ | ✅ | **✅ (按时间/大小自动删除)** |
| **压缩** | Gorilla + RLE | 列压缩 | PG TOAST | 列压缩 | **✅（冷数据分片压缩）** |
| **集群/分布式** | ✅ (企业版) | ✅ (原生) | ❌ (靠 PG 方案) | ✅ (企业版) | **✅（集群 + 主从同步）** |
| **MQ 能力** | ❌ | ❌ | ❌ | ❌ | **✅ (内置 Stream/MQ)** |
| **运维复杂度** | 中 | 中偏高 | 低 (PG 生态) | 低 | **极低（文件夹即库）** |
| **适用规模** | 中大型 | 中大型 | 中大型 | 中大型 | **中大型/嵌入** |
| **.NET 集成** | HTTP SDK | 原生驱动 | Npgsql | HTTP / PGWire | **原生嵌入 + ADO.NET** |

## 4. NoSQL 数据库对比

| 特性 | Redis | MongoDB | LevelDB / RocksDB | etcd | **NovaDb (KV 模式)** |
| :--- | :--- | :--- | :--- | :--- | :--- |
| **实现语言** | C | C++ | C++ | Go | **C#** |
| **数据模型** | KV + 多种数据结构 | 文档 (BSON) | 有序 KV | KV (强一致) | **KV 表 (Key + Value + TTL)** |
| **部署方式** | 独立服务 / 集群 | 独立服务 / 副本集 | 嵌入式库 | 独立集群 (3/5 节点) | **嵌入 / 独立服务** |
| **持久化** | RDB / AOF (可选) | WiredTiger (默认持久) | WAL + SSTable | WAL + Snapshot | **WAL + MMF (强持久化)** |
| **事务** | MULTI/EXEC (弱事务) | 多文档事务 (4.0+) | 批量写 (原子) | Mini 事务 (Revision) | **MVCC 事务 (Read Committed)** |
| **查询能力** | 命令式 (无 SQL) | 丰富查询 + 聚合管道 | Get/Put/Scan | Range + Watch | **SQL 查询 + API** |
| **内存模型** | 全内存为主 | 工作集缓存 (WiredTiger) | 块缓存 (Block Cache) | 全内存 (BoltDB) | **冷热分离 (热段内存+冷段 MMF)** |
| **数据结构** | String/Hash/List/Set/ZSet/Stream | 文档/数组/嵌套 | 有序字节 KV | 扁平 KV | **Key + Value + TTL（每行）** |
| **TTL 支持** | ✅ (Key 级别) | ✅ (TTL 索引) | ❌ (需自实现) | ✅ (Lease) | **✅（每行 TTL）** |
| **发布订阅** | ✅ (Pub/Sub + Stream) | ✅ (Change Stream) | ❌ | ✅ (Watch) | **✅ (Flux Engine MQ)** |
| **分布式** | Cluster / Sentinel | 副本集 / 分片 | ❌ (嵌入式) | Raft 共识 (强一致) | **✅（集群）** |
| **运维复杂度** | 低 | 中 | 极低 (嵌入) | 中 | **极低（文件夹即库）** |
| **适用规模** | 大型/超大型 | 大型/超大型 | 嵌入式 | 大型 | **大型/嵌入式** |
| **适用场景** | 缓存/会话/排行榜/队列 | 文档存储/内容管理 | 嵌入式存储/LSM 底层 | 配置中心/服务发现 | **缓存/会话/配置/分布式锁** |
| **.NET 集成** | StackExchange.Redis | MongoDB.Driver | 需 P/Invoke | dotnet-etcd | **原生嵌入 + ADO.NET** |

## 6. NovaDb 综合差异化总结

| 差异化能力 | 说明 |
| :--- | :--- |
| **纯 .NET 托管** | 无 Native 依赖，支持 .NET Framework 4.5+，部署即 xcopy，与 .NET 应用同进程零序列化开销 |
| **嵌入+服务双模** | 开发调试嵌入如 SQLite，生产部署独立服务如 MySQL，同一套 API |
| **集群与主从同步** | 独立服务支持集群部署，支持一主多从同步，用于读扩展与容灾；多主多从作为远期目标 |
| **文件夹即数据库** | 拷贝文件夹完成迁移/备份，无需 dump/restore 流程 |
| **冷热分离索引** | 1000 万行表仅查热点时内存 < 20MB，冷数据自动卸载至 MMF |
| **关系型 + 时序 + MQ + KV 融合** | 单一引擎覆盖 SQLite + TDengine + Redis 常见场景，减少组件数 |
| **NewLife XCode 原生集成** | ORM 层直接适配，无需第三方驱动 |
