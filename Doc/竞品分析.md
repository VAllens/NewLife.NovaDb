# NewLife.NovaDb 竞品分析

> 帮助团队与用户回答"为什么选 NovaDb / 不选 NovaDb"。

---

## 1. 一句话定位

NovaDb 是一个**纯 .NET 托管**、**嵌入式/网络双模**的混合数据库，将**关系型 + 时序 + MQ + KV** 融合在同一引擎，优先追求**工程集成效率与低运维成本**。

---

## 2. 关系型对比（SQLite / MySQL / PostgreSQL）

| 维度 | SQLite | MySQL | PostgreSQL | **NovaDb** |
|------|--------|-------|-----------|-----------|
| **实现语言** | C | C/C++ | C | **C#** |
| **运行形态** | 嵌入（库） | 独立服务 | 独立服务 | **嵌入 / 服务** |
| **部署** | 单文件 | 安装配置 | 安装配置 | **文件夹即数据库** |
| **索引** | B+ 树（磁盘） | B+ 树（磁盘） | B+ 树 / GIN / GiST | **SkipList（内存+冷热分离）** |
| **事务隔离** | Serializable | RC / RR | RC / RR / Serializable | **Read Committed（MVCC）** |
| **并发** | 单写多读 | 多写多读 | 多写多读 | **MVCC 多写多读 + 嵌入表级锁** |
| **备份** | 拷贝文件 | mysqldump | pg_dump | **拷贝文件夹** |
| **扩展能力** | 纯关系型 | 纯关系型 | 关系型+JSON | **关系型 + 时序 + MQ + KV** |
| **SQL 完整度** | ~85% | ~95% | ~98% | **~60%（业务常用子集）** |
| **跨平台** | ✅ | ✅ | ✅ | **✅（.NET 跨平台）** |

### NovaDb 的取舍

- **取**：常用 SQL + ADO.NET + 工程集成，嵌入/网络双模，多引擎融合
- **舍**：不追求 SQL 全功能（无窗口函数/CTE/触发器/存储过程）

### SQL 能力边界（v1）

| SQL 能力 | SQLite | MySQL | PG | **NovaDb v1** |
|----------|--------|-------|-----|-------------|
| CREATE/ALTER/DROP | ✅ | ✅ | ✅ | **✅（P0）** |
| INSERT/UPDATE/DELETE | ✅ | ✅ | ✅ | **✅（P0）** |
| UPSERT | ✅ | ✅ | ✅ | **✅（P0）** |
| SELECT/WHERE/ORDER BY | ✅ | ✅ | ✅ | **✅（P0）** |
| GROUP BY/HAVING/聚合 | ✅ | ✅ | ✅ | **✅（P0）** |
| JOIN | ✅ | ✅ | ✅ | **⚠️ 基础（P1）** |
| 子查询 | ✅ | ✅ | ✅ | **⚠️ 基础（P1）** |
| CTE/窗口函数 | ✅ | ✅ | ✅ | **❌** |
| 视图/触发器/存储过程 | 部分 | ✅ | ✅ | **❌** |
| 外键 | ✅ | ✅ | ✅ | **❌** |

## 3. 时序对比（InfluxDB / TDengine / TimescaleDB）

| 维度 | InfluxDB | TDengine | TimescaleDB | **NovaDb Flux** |
|------|----------|----------|------------|----------------|
| **语言** | Go/Rust | C | C (PG) | **C#** |
| **部署** | 独立服务 | 独立/集群 | PG 扩展 | **嵌入 / 服务** |
| **写入** | Line Protocol | 追加写 | SQL INSERT | **追加写 + 分片** |
| **降采样** | 连续查询 | 内置 | Toolkit | **规划支持** |
| **TTL** | ✅ | ✅ | ✅ | **✅** |
| **时间窗口聚合** | ✅ | ✅ | ✅ | **✅** |
| **MQ 融合** | ❌ | ❌ | ❌ | **✅（内置 Stream/MQ）** |
| **消息可靠性** | ❌ | ❌ | ❌ | **✅（At-Least-Once）** |
| **.NET 集成** | HTTP SDK | 原生驱动 | Npgsql | **原生嵌入 + ADO.NET** |

**NovaDb Flux 特色**：内置 MQ + 时序融合，单引擎替代"时序库 + 消息队列"组合。

---

## 4. MQ 对比（Kafka / RabbitMQ / Redis Stream）

| 维度 | Kafka | RabbitMQ | Redis Stream | **NovaDb MQ** |
|------|-------|----------|-------------|---------------|
| **部署** | 高（ZK/KRaft） | 中（Erlang） | 低 | **极低** |
| **消息模型** | 分区+消费组 | Exchange+Queue | Stream+消费组 | **Stream+消费组** |
| **可靠性** | ALO / EO | ALO / AMO | ALO | **ALO** |
| **顺序** | 分区内有序 | 单队列有序 | 全局有序 | **全局有序** |
| **延迟消息** | ❌ | ✅ | ❌ | **✅** |
| **死信队列** | ❌ | ✅ | ❌ | **✅** |
| **SQL 查询** | ❌（KSQL 独立） | ❌ | ❌ | **✅（原生 SQL）** |
| **吞吐** | 极高（百万/s） | 中（万级） | 高（十万/s） | **中等** |

NovaDb 适合"内部任务队列/事件流水/轻量消息"，不适合大规模多租户消息平台。

---

## 5. KV / NoSQL 对比（Redis / MongoDB / etcd）

| 维度 | Redis | MongoDB | etcd | **NovaDb KV** |
|------|-------|---------|------|-------------|
| **数据模型** | KV + 多种结构 | 文档（BSON） | KV（强一致） | **KV 表 + TTL** |
| **存储模型** | 全内存 | WiredTiger | BoltDB | **Bitcask（索引内存+值磁盘）** |
| **部署** | 独立/集群 | 独立/副本集 | 集群 3/5 节点 | **嵌入 / 服务** |
| **内存** | 全内存 | 工作集缓存 | 全内存 | **~20B/key 索引，值按需读取** |
| **TTL** | ✅ Key 级 | ✅ TTL 索引 | ✅ Lease | **✅ 每行 TTL + 惰性/主动清理** |
| **事务** | 弱 | 多文档 4.0+ | Mini 事务 | **MVCC（RC）** |
| **原子操作** | INCR/SETNX | FindOneAndUpdate | Txn | **Add/Inc/IncDouble/Replace** |
| **查询** | 命令式 | 聚合管道 | Range+Watch | **SQL + API** |
| **自动压缩** | 不需要 | 后台 Compact | 自动 | **自动 Compact（比率触发）** |
| **.NET 集成** | StackExchange | MongoDB.Driver | dotnet-etcd | **原生嵌入** |

**NovaDb KV 特色**：Bitcask 模型实现低内存占用（百万键 ~20MB），多表隔离、SQL + API 双访问、MVCC 事务、自动压缩。

## 6. OLAP / 物化视图对比

| 维度 | ClickHouse | TimescaleDB | **NovaDb 物化视图** |
|------|-----------|------------|-------------------|
| **物化视图** | ✅ 原生 | ✅ 连续聚合 | **✅ 增量物化视图** |
| **刷新** | INSERT 触发 | 定时+实时 | **增量更新+定时修正** |
| **聚合状态** | ✅ State 函数 | ❌ 直接存结果 | **✅ State 函数** |
| **MQ 消费聚合** | ❌ 需 Kafka | ❌ 需外部 MQ | **✅ 内置 MQ+聚合** |
| **多粒度汇总** | ✅ 手动 | ✅ time_bucket | **✅ 分层聚合** |

NovaDb 不替代专业 OLAP；强调"把常用聚合收进同一引擎"。

## 7. 场景适配度

| 场景 | 适配度 | 关键原因 | 备选 |
|------|:------:|---------|------|
| **.NET 嵌入式应用** | ⭐⭐⭐⭐⭐ | 纯托管、零部署、同进程 | SQLite |
| **小型 .NET 服务** | ⭐⭐⭐⭐⭐ | 文件夹即库、低运维 | MySQL/PG |
| **IoT 数据采集** | ⭐⭐⭐⭐ | 时序+MQ、自动 TTL | TDengine/Kafka |
| **消息队列** | ⭐⭐⭐⭐ | 顺序、Pending+Ack、SQL 查询 | Redis Stream |
| **会话/缓存** | ⭐⭐⭐⭐ | KV+TTL、冷热分离 | Redis |
| **实时统计** | ⭐⭐⭐⭐⭐ | 增量物化视图+MQ 聚合 | ClickHouse |
| **中大型 OLTP** | ⭐⭐⭐⭐ | 集群、MVCC | MySQL/PG |
| **高级 SQL** | ❌ | 无窗口函数/CTE | PG/SQL Server |
| **超大 TPS** | ❌ | .NET GC 限制 | MySQL/Oracle |
| **专业 OLAP** | ❌ | 列存储能力有限 | ClickHouse |
| **大规模 MQ** | ❌ | 单机吞吐限制 | Kafka/Pulsar |

### 选型决策流程

1. 是 .NET 项目吗？→ 否则考虑其他方案
2. 能接受嵌入或单机服务化？→ 否则需成熟分布式 RDBMS
3. 主要是 OLTP 还是 OLAP？→ OLAP 需结合物化视图或外接专业库
4. 需要高级 SQL 特性？→ 是则 PG/SQL Server 更合适

## 8. 差异化总结

| 差异化能力 | 说明 |
|-----------|------|
| **纯 .NET 托管** | 无 Native 依赖，.NET 4.5+，xcopy 部署 |
| **嵌入+网络双模** | 开发嵌入如 SQLite，生产网络如 MySQL |
| **文件夹即数据库** | 拷贝文件夹完成迁移/备份 |
| **冷热分离索引** | 热点内存 < 20MB，冷数据自动卸载 |
| **四引擎融合** | 关系型+时序+MQ+KV 一套引擎 |
| **增量物化视图** | 时序实时聚合+定时修正 |
| **XCode 原生集成** | ORM 直接适配 |
| **明确非目标** | 不与全功能 SQL / 专业 OLAP / 分布式强一致竞争 |
