# NewLife.NovaDb 用户指南

> NovaDb 是一个纯 C# 实现的混合数据库引擎，支持嵌入式/服务器双模式，融合关系型、时序、消息队列、KV 四大能力。无需外部依赖，开箱即用。

---

## 一、快速开始

### 1.1 安装

通过 NuGet 安装 NovaDb 核心包：

```shell
dotnet add package NewLife.NovaDb
```

### 1.2 嵌入式模式（5 分钟上手）

嵌入式模式无需启动独立服务，适合桌面应用、IoT 设备、单元测试等场景。连接字符串以 `Data Source=` 指定本地数据库路径，文件夹即数据库。

```csharp
using NewLife.NovaDb.Client;

// 创建连接（嵌入式模式）
using var conn = new NovaConnection { ConnectionString = "Data Source=./mydb" };
conn.Open();

// 建表
using var cmd = conn.CreateCommand();
cmd.CommandText = "CREATE TABLE users (id INT PRIMARY KEY, name STRING, age INT)";
cmd.ExecuteNonQuery();

// 插入数据
cmd.CommandText = "INSERT INTO users (id, name, age) VALUES (1, 'Alice', 25)";
cmd.ExecuteNonQuery();

// 批量插入
cmd.CommandText = @"INSERT INTO users (id, name, age) VALUES 
    (2, 'Bob', 30), 
    (3, 'Charlie', 28)";
cmd.ExecuteNonQuery();

// 查询数据
cmd.CommandText = "SELECT * FROM users WHERE age >= 25 ORDER BY age";
using var reader = cmd.ExecuteReader();
while (reader.Read())
{
    Console.WriteLine($"id={reader["id"]}, name={reader["name"]}, age={reader["age"]}");
}

// 更新数据
cmd.CommandText = "UPDATE users SET age = 26 WHERE id = 1";
cmd.ExecuteNonQuery();

// 删除数据
cmd.CommandText = "DELETE FROM users WHERE id = 3";
cmd.ExecuteNonQuery();
```

连接字符串还支持 WAL 模式和只读模式：

```csharp
// 高性能模式（不写 WAL）
var cs = "Data Source=./mydb;WalMode=None";

// 只读模式
var cs = "Data Source=./mydb;ReadOnly=true";
```

### 1.3 服务器模式

服务器模式通过 TCP 提供远程访问，支持多客户端并发连接，适合分布式部署。

**启动服务端：**

```csharp
using NewLife.NovaDb.Server;

var svr = new NovaServer(3306) { DbPath = "./data" };

// 可选：自定义配置
svr.Options = new ServerDbOptions
{
    WalMode = WalMode.Normal,
    EnableBinlog = true,
    PageCacheSize = 2048,
};

svr.Start();
Console.WriteLine("NovaDb 服务已启动，端口: 3306");
Console.ReadLine();
svr.Stop("手动关闭");
```

**ADO.NET 客户端连接：**

```csharp
using NewLife.NovaDb.Client;

// 通过连接字符串中的 Server 参数自动切换到服务器模式
using var conn = new NovaConnection { ConnectionString = "Server=127.0.0.1;Port=3306" };
conn.Open();

// 与嵌入式模式使用完全相同的 ADO.NET API
using var cmd = conn.CreateCommand();
cmd.CommandText = "SELECT * FROM users";
using var reader = cmd.ExecuteReader();
while (reader.Read())
{
    Console.WriteLine($"name={reader["name"]}");
}
```

**RPC 客户端连接：**

```csharp
using NewLife.NovaDb.Client;

// 直接使用 NovaClient 进行 RPC 调用
using var client = new NovaClient("tcp://127.0.0.1:3306");
client.Open();

// 执行查询
var result = await client.QueryAsync<IDictionary<String, Object>[]>("SELECT * FROM users");

// 执行非查询语句
var affected = await client.ExecuteAsync("INSERT INTO users (id, name) VALUES (10, 'Dave')");

// 心跳检测
var serverTime = await client.PingAsync();
Console.WriteLine($"服务器时间: {serverTime}");
```

## 二、核心概念

### 2.1 文件夹即数据库

NovaDb 采用**文件夹即数据库**的设计理念，每个数据库对应文件系统中的一个目录。

```
mydb/                        # 数据库根目录
├── users.data               # users 表的数据文件
├── users.idx                # users 表的索引文件
├── users.wal                # users 表的 WAL 文件
├── orders.data
├── orders.idx
├── orders.wal
└── flux/                    # 时序数据分区目录
    ├── metrics_2025010100/  # 按小时分区
    └── metrics_2025010101/
```

这种设计带来以下优势：
- **零配置**：指定路径即可使用，无需安装数据库服务
- **轻松迁移**：复制文件夹即可完成数据库迁移或备份
- **独立隔离**：不同数据库互不影响

### 2.2 WAL 模式

WAL（Write-Ahead Logging）决定了数据写入时的持久化策略，通过 `WalMode` 枚举配置：

| 模式 | 说明 | 适用场景 |
|------|------|---------|
| `Full` | 每次写入同步刷盘，确保数据不丢失 | 金融、订单等关键业务 |
| `Normal`（默认） | 异步刷盘，每秒写入一次 | 大多数业务场景 |
| `None` | 不写 WAL，最高吞吐 | 缓存、临时数据、可重建数据 |

```csharp
var options = new DbOptions
{
    Path = "./mydb",
    WalMode = WalMode.Full,  // 最安全模式
};
```

### 2.3 数据类型

NovaDb 支持以下数据类型，覆盖常见业务场景和 AI 应用：

| 类型 | SQL 关键字 | 存储大小 | 说明 |
|------|-----------|---------|------|
| 布尔 | `BOOLEAN` | 1 字节 | true/false |
| 整数 | `INT` | 4 字节 | 32 位有符号整数 |
| 长整数 | `LONG` / `BIGINT` | 8 字节 | 64 位有符号整数 |
| 浮点数 | `DOUBLE` | 8 字节 | 双精度浮点 |
| 高精度 | `DECIMAL` | 16 字节 | 财务等高精度计算 |
| 日期时间 | `DATETIME` | 8 字节 | Int64 Ticks（100 纳秒），0001-9999 年 |
| 字符串 | `STRING` | 变长 | UTF-8 编码 |
| 二进制 | `BINARY` / `BLOB` | 变长 | 字节数组 |
| 地理坐标 | `GEOPOINT` | 16 字节 | 经纬度坐标（lat/lon） |
| 向量 | `VECTOR(n)` | 4×n 字节 | 浮点数组，用于 AI 向量检索 |

## 四、事务

NovaDb 基于 MVCC（多版本并发控制）实现事务隔离，默认隔离级别为 **Read Committed**。

### 4.1 ADO.NET 事务

```csharp
using var conn = new NovaConnection { ConnectionString = "Data Source=./mydb" };
conn.Open();

// 开启事务
using var tx = conn.BeginTransaction();
try
{
    using var cmd = conn.CreateCommand();
    cmd.Transaction = tx;

    // 扣减库存
    cmd.CommandText = "UPDATE products SET stock = stock - 1 WHERE id = 1 AND stock > 0";
    var affected = cmd.ExecuteNonQuery();
    if (affected == 0) throw new InvalidOperationException("库存不足");

    // 创建订单
    cmd.CommandText = "INSERT INTO orders (id, product_id, amount) VALUES (1001, 1, 1)";
    cmd.ExecuteNonQuery();

    tx.Commit();
}
catch
{
    tx.Rollback();
    throw;
}
```

### 4.2 RPC 事务

通过 `NovaClient` 使用事务，事务 ID 由服务端生成并返回：

```csharp
using var client = new NovaClient("tcp://127.0.0.1:3306");
client.Open();

// 开启事务
var txId = await client.BeginTransactionAsync();
try
{
    await client.ExecuteAsync("UPDATE products SET stock = stock - 1 WHERE id = 1");
    await client.ExecuteAsync("INSERT INTO orders (id, product_id) VALUES (1001, 1)");

    // 提交事务
    await client.CommitTransactionAsync(txId);
}
catch
{
    // 回滚事务
    await client.RollbackTransactionAsync(txId);
    throw;
}
```

### 4.3 隔离级别与 MVCC

NovaDb 采用 MVCC 机制实现并发控制：

- **默认隔离级别**：Read Committed，每次读取都能看到已提交的最新数据
- **行版本**：每行数据附带版本号（`RowVersion`），写入时生成新版本
- **无锁读取**：读操作不阻塞写操作，写操作不阻塞读操作
- **冲突检测**：同一行的并发写入通过版本号检测冲突

## 五、时序引擎（Flux）

Flux 引擎专为时间序列数据设计，采用 Append-Only 追加写入和按时间自动分区策略，适合物联网监控、日志采集等场景。

### 5.1 创建时序表

```sql
-- 时序表使用 FLUX 引擎
CREATE TABLE metrics (
    timestamp DATETIME,
    device_id STRING,
    temperature DOUBLE,
    humidity DOUBLE
) ENGINE=FLUX;
```

### 5.2 写入数据

通过 `FluxEngine` API 直接写入：

```csharp
using NewLife.NovaDb.Engine.Flux;

var engine = new FluxEngine("./data/metrics", new DbOptions
{
    FluxPartitionHours = 1,        // 每小时一个分区
    FluxDefaultTtlSeconds = 86400, // 数据保留 24 小时
});

// 写入单条数据
var entry = new FluxEntry
{
    Timestamp = DateTime.UtcNow.Ticks,
    Tags = new Dictionary<String, String>
    {
        ["device"] = "sensor-001",
        ["location"] = "北京",
    },
    Fields = new Dictionary<String, Object?>
    {
        ["temperature"] = 23.5,
        ["humidity"] = 65.0,
    },
};
engine.Append(entry);

// 批量写入
var batch = new List<FluxEntry>();
for (var i = 0; i < 1000; i++)
{
    batch.Add(new FluxEntry
    {
        Timestamp = DateTime.UtcNow.AddSeconds(-i).Ticks,
        Tags = new Dictionary<String, String> { ["device"] = $"sensor-{i % 10:D3}" },
        Fields = new Dictionary<String, Object?> { ["value"] = Random.Shared.NextDouble() * 100 },
    });
}
engine.AppendBatch(batch);
```

### 5.3 时间范围查询

```csharp
// 查询最近一小时的数据
var startTicks = DateTime.UtcNow.AddHours(-1).Ticks;
var endTicks = DateTime.UtcNow.Ticks;

var results = engine.QueryRange(startTicks, endTicks);
foreach (var e in results)
{
    var ts = new DateTime(e.Timestamp, DateTimeKind.Utc);
    Console.WriteLine($"[{ts:HH:mm:ss}] device={e.Tags["device"]}, temp={e.Fields["temperature"]}");
}
```

### 5.4 分区管理

Flux 引擎按 `FluxPartitionHours` 自动创建时间分区，分区键格式为 `yyyyMMddHH`。

```csharp
// 查看分区数量
var count = engine.GetPartitionCount();
Console.WriteLine($"当前分区数: {count}");

// 查看总数据量
var total = engine.GetEntryCount();
Console.WriteLine($"总记录数: {total}");

// 获取分区键
var key = engine.GetPartitionKey(DateTime.UtcNow.Ticks);
Console.WriteLine($"当前分区: {key}");  // 例如: 2025070112

// 清理过期分区（保留最近 7 天）
var deleted = engine.DeleteExpiredPartitions(7 * 24 * 3600);
Console.WriteLine($"已清理 {deleted} 个过期分区");
```

## 六、消息队列（MQ）

NovaDb 基于 Flux 时序引擎实现了 Redis Stream 风格的消息队列，通过 `StreamManager` 提供发布/消费/确认/死信等完整功能。

### 6.1 发布消息

```csharp
using NewLife.NovaDb.Engine.Flux;

var stream = new StreamManager(fluxEngine);

// 发布消息
var entry = new FluxEntry
{
    Timestamp = DateTime.UtcNow.Ticks,
    Fields = new Dictionary<String, Object?>
    {
        ["orderId"] = 10001,
        ["action"] = "created",
        ["amount"] = 299.00,
    },
};
var msgId = stream.Publish(entry);
Console.WriteLine($"消息已发布: {msgId}");
```

### 6.2 消费组

```csharp
// 创建消费组
stream.CreateConsumerGroup("order-processor");

// 消费者读取消息（同步）
var messages = stream.ReadGroup("order-processor", "worker-1", count: 10);
foreach (var msg in messages)
{
    Console.WriteLine($"收到消息: {msg.GetMessageId()}, orderId={msg.Fields["orderId"]}");
}

// 异步阻塞读取（等待新消息，超时 30 秒）
var msgs = await stream.ReadGroupAsync(
    "order-processor", "worker-1",
    count: 10,
    timeout: TimeSpan.FromSeconds(30));
```

### 6.3 消息确认

消费者处理完消息后必须确认（ACK），未确认的消息会进入 Pending 列表：

```csharp
foreach (var msg in messages)
{
    try
    {
        ProcessOrder(msg);

        // 确认消息
        var (ts, seq) = FluxEntry.ParseMessageId(msg.GetMessageId());
        stream.Acknowledge("order-processor", new MessageId(ts, seq));
    }
    catch (Exception ex)
    {
        Console.WriteLine($"处理失败: {ex.Message}");
    }
}

// 查看未确认的消息
var pending = stream.GetPendingEntries("order-processor");
Console.WriteLine($"待处理消息: {pending.Count} 条");
```

### 6.4 死信队列

处理多次失败的消息会被转移到死信队列（DLQ），便于后续排查和重试：

```csharp
// 手动将消息移入死信队列
stream.MoveToDeadLetter("order-processor", msgId, "处理超时");

// 自动处理过期的 Pending 消息（超过 5 分钟未确认则进入死信）
var moved = stream.ProcessExpiredPending("order-processor", TimeSpan.FromMinutes(5));
Console.WriteLine($"已移入死信队列: {moved} 条");

// 读取死信消息
var deadLetters = stream.ReadDeadLetters("order-processor", count: 50);
foreach (var dl in deadLetters)
{
    Console.WriteLine($"死信: {dl.MessageId}, 原因: {dl.Reason}");
}

// 重试死信消息
stream.RetryDeadLetter("order-processor", deadMsgId);

// 删除死信消息（确认不需要重试）
stream.DeleteDeadLetter("order-processor", deadMsgId);
```

### 6.5 延迟消息

支持延迟发布和定时发布，适合订单超时、定时通知等场景：

```csharp
// 延迟 30 分钟发布（订单超时取消）
var entry = new FluxEntry
{
    Timestamp = DateTime.UtcNow.Ticks,
    Fields = new Dictionary<String, Object?> { ["orderId"] = 10001, ["action"] = "timeout_check" },
};
stream.PublishDelayed(entry, TimeSpan.FromMinutes(30));

// 定时发布（指定具体时间）
stream.PublishScheduled(entry, DateTime.UtcNow.AddHours(2));

// 需要定期调用 DeliverDueMessages 投递到期的延迟消息
var delivered = stream.DeliverDueMessages();
Console.WriteLine($"已投递 {delivered} 条延迟消息");

// 查看待投递的延迟消息数量
var delayedCount = stream.GetDelayedMessageCount();
Console.WriteLine($"等待投递: {delayedCount} 条");
```

## 七、KV 存储

KvStore 提供高性能的键值存储，支持 TTL 自动过期、原子递增、分布式锁等功能。

### 7.1 基本操作

```csharp
using NewLife.NovaDb.Engine.KV;

var kv = new KvStore("./data/cache", new DbOptions
{
    DefaultKvTtl = TimeSpan.FromHours(24),  // 默认过期时间 24 小时
});

// 写入二进制数据
kv.Set("session:1001", Encoding.UTF8.GetBytes("user-data"), TimeSpan.FromMinutes(30));

// 读取数据
var val = kv.Get("session:1001");
if (val != null)
    Console.WriteLine($"数据: {Encoding.UTF8.GetString(val)}");

// 检查是否存在
if (kv.Exists("session:1001"))
    Console.WriteLine("会话有效");

// 删除数据
kv.Delete("session:1001");

// 获取所有 Key
var keys = kv.GetAllKeys();
Console.WriteLine($"共 {kv.Count} 个有效键");
```

### 7.2 TTL 过期

```csharp
// 写入时指定 TTL
kv.Set("token:abc", data, TimeSpan.FromHours(2));

// 查看剩余过期时间
var expireAt = kv.GetExpiration("token:abc");
if (expireAt.HasValue)
    Console.WriteLine($"过期时间: {expireAt.Value:yyyy-MM-dd HH:mm:ss}");

// 更新 TTL（续期）
kv.SetExpiration("token:abc", TimeSpan.FromHours(4));

// 手动清理所有过期数据
var cleaned = kv.CleanupExpired();
Console.WriteLine($"已清理 {cleaned} 个过期键");
```

### 7.3 原子递增

```csharp
// 原子递增（适合计数器、限流等场景）
var count = kv.Inc("page:views:/index", 1, TimeSpan.FromDays(1));
Console.WriteLine($"今日访问量: {count}");

// 递减
var stock = kv.Inc("stock:product:1001", -1);
Console.WriteLine($"剩余库存: {stock}");
```

### 7.4 字符串便捷方法

```csharp
// 直接操作字符串（内部自动 UTF-8 编解码）
kv.SetString("config:app:name", "我的应用", TimeSpan.FromDays(365));
var name = kv.GetString("config:app:name");
Console.WriteLine($"应用名称: {name}");

// Add 方法：仅在 Key 不存在时写入（可用于分布式锁）
var locked = kv.AddString("lock:order:1001", "worker-1", TimeSpan.FromSeconds(30));
if (locked)
{
    try
    {
        // 获得锁，执行业务逻辑
        ProcessOrder(1001);
    }
    finally
    {
        kv.Delete("lock:order:1001");
    }
}
else
{
    Console.WriteLine("未能获取锁，其他节点正在处理");
}
```

## 八、集群部署

NovaDb 支持主从架构的集群部署，通过 Binlog 实现数据同步。

### 8.1 主从架构

```
┌──────────┐    Binlog 同步    ┌──────────┐
│  主节点   │ ──────────────→  │  从节点 1  │
│  (读写)   │                  │  (只读)    │
└──────────┘                  └──────────┘
      │         Binlog 同步    ┌──────────┐
      └──────────────────────→ │  从节点 2  │
                               │  (只读)    │
                               └──────────┘
```

### 8.2 配置主节点

```csharp
using NewLife.NovaDb.Server;

var svr = new NovaServer(3306) { DbPath = "./data" };
svr.Options = new ServerDbOptions
{
    EnableBinlog = true,             // 启用 Binlog（主从同步必须开启）
    BinlogMaxFileSize = 256 * 1024 * 1024,  // 单个 Binlog 文件最大 256MB
    BinlogRetentionDays = 7,         // Binlog 保留 7 天
    WalMode = WalMode.Full,          // 建议主节点使用 Full 模式
};
svr.Start();
```

### 8.3 配置从节点

```csharp
using NewLife.NovaDb.Cluster;

// 定义本节点信息
var localNode = new NodeInfo { NodeId = "slave-1", Endpoint = "192.168.1.11:3306" };

// 连接主节点
var replica = new ReplicaClient(localNode, "192.168.1.10:3306", (lsn, data) =>
{
    // 回调：应用主节点同步过来的数据
    Console.WriteLine($"应用 LSN={lsn}, 数据大小={data.Length} 字节");
});

replica.Connect();

// 获取断点续传位置
var resumePos = replica.GetResumePosition();
Console.WriteLine($"从 LSN={resumePos} 开始同步");
```

### 8.4 数据同步

主节点通过 `ReplicationManager` 管理从节点的同步状态：

```csharp
using NewLife.NovaDb.Cluster;

// 注册从节点
var slave = new NodeInfo { NodeId = "slave-1", Endpoint = "192.168.1.11:3306" };
replicationManager.RegisterSlave(slave);

// 获取从节点同步进度
var lag = replicationManager.GetReplicationLag("slave-1");
Console.WriteLine($"从节点延迟: {lag} 条记录");

// 获取待同步的记录
var pending = replicationManager.GetPendingRecords("slave-1", maxCount: 100);

// 确认同步完成
replicationManager.AcknowledgeReplication("slave-1", lastLsn);

// 检查是否所有从节点已同步到指定位置
var synced = replicationManager.IsFullySynced(currentLsn);

// 清理已同步的缓冲区
var cleaned = replicationManager.CleanupBuffer();

// 移除从节点
replicationManager.RemoveSlave("slave-1");

// 列出所有从节点
var slaves = replicationManager.GetAllSlaves();
foreach (var s in slaves)
{
    Console.WriteLine($"节点: {s.NodeId}, 地址: {s.Endpoint}");
}
```

## 九、性能优化

### 9.1 索引冷热分离

NovaDb 自动将高频访问的索引保留在内存（热索引），低频索引卸载到磁盘（冷索引），通过 `HotIndexManager` 和 `ColdIndexDirectory` 实现。

```csharp
var options = new DbOptions
{
    HotWindowSeconds = 600,      // 热窗口：最近 10 分钟访问过的索引保留在内存
    ColdEvictionSeconds = 1800,  // 冷驱逐：30 分钟未访问的索引卸载到磁盘
    PageCacheSize = 2048,        // 页面缓存大小（页数）
};
```

### 9.2 分片策略

当单表数据量超过阈值时，NovaDb 通过 `ShardManager` 自动将数据拆分到多个分片：

```csharp
var options = new DbOptions
{
    ShardSizeThreshold = 1L * 1024 * 1024 * 1024,  // 单分片超过 1GB 时触发分片
    ShardRowThreshold = 10_000_000,                  // 单分片超过 1000 万行时触发分片
};
```

### 9.3 批量写入

批量操作比逐条操作性能提升数倍，建议在大量数据写入时使用批量 SQL 或批量 API：

```csharp
// 方式一：批量 SQL
var sb = new StringBuilder("INSERT INTO logs (id, message, created) VALUES ");
for (var i = 0; i < 10000; i++)
{
    if (i > 0) sb.Append(',');
    sb.Append($"({i}, 'log-{i}', '{DateTime.Now:yyyy-MM-dd HH:mm:ss}')");
}
cmd.CommandText = sb.ToString();
cmd.ExecuteNonQuery();

// 方式二：事务包裹（减少 WAL 刷盘次数）
using var tx = conn.BeginTransaction();
using var cmd2 = conn.CreateCommand();
cmd2.Transaction = tx;
for (var i = 0; i < 10000; i++)
{
    cmd2.CommandText = $"INSERT INTO logs (id, message) VALUES ({i}, 'log-{i}')";
    cmd2.ExecuteNonQuery();
}
tx.Commit();

// 方式三：FluxEngine 批量追加
engine.AppendBatch(entries);
```

### 9.4 WAL 模式选择

根据业务场景选择合适的 WAL 模式：

| 场景 | 推荐模式 | 原因 |
|------|---------|------|
| 金融交易、订单 | `Full` | 数据绝对不能丢失 |
| 普通业务、用户数据 | `Normal` | 最多丢失 1 秒数据，性能与安全平衡 |
| 缓存、临时计算、日志 | `None` | 最高写入吞吐，可接受崩溃丢数据 |
| 物联网传感器数据 | `Normal` | 数据量大，偶尔丢失可接受 |
| 主节点（有从节点） | `Full` | 确保 Binlog 完整性 |

| `ShardSizeThreshold` | `Int64` | `1073741824` | 分片大小阈值（字节，默认 1GB） |

## 十一、常见问题

### Q1：嵌入式模式和服务器模式如何选择？

| 场景 | 推荐模式 |
|------|---------|
| 桌面应用、移动端、IoT 设备 | 嵌入式模式 |
| 单元测试、原型开发 | 嵌入式模式 |
| 微服务、Web 应用 | 服务器模式 |
| 多进程/多机访问同一数据库 | 服务器模式 |

### Q2：如何备份数据库？

直接复制数据库文件夹即可完成备份：

```shell
# 停机备份（最安全）
cp -r ./mydb ./mydb_backup_$(date +%Y%m%d)

# 只读模式打开进行在线备份
```

### Q3：数据库文件损坏如何恢复？

NovaDb 在启动时会自动通过 WAL 文件（`WalRecovery`）进行崩溃恢复。如果开启了数据校验和（`EnableChecksum=true`），还会检测并报告损坏的页面。

### Q4：如何提升写入性能？

1. 使用 `WalMode=None`（可接受丢数据时）
2. 使用批量插入代替逐条插入
3. 将多次写入包裹在同一个事务中
4. 增大 `PageCacheSize` 减少磁盘 IO
5. 时序数据使用 `FluxEngine.AppendBatch()` 批量追加

### Q5：NovaDb 支持哪些 .NET 版本？

NovaDb 支持 .NET Framework 4.5 到 .NET 10 的全部版本，通过条件编译确保兼容性。

### Q6：如何监控数据库状态？

```csharp
// 嵌入式模式
Console.WriteLine($"是否嵌入式: {conn.IsEmbedded}");
Console.WriteLine($"数据库: {conn.Database}");
Console.WriteLine($"数据源: {conn.DataSource}");

// 服务器模式
var serverTime = await client.PingAsync();
Console.WriteLine($"服务器连接正常，时间: {serverTime}");
Console.WriteLine($"是否连接: {client.IsConnected}");
```

### Q7：消费组中的消息一直未被确认怎么办？

使用 `ProcessExpiredPending` 方法自动将超时未确认的消息移入死信队列：

```csharp
// 每分钟执行一次，将 5 分钟未确认的消息移入死信
var moved = stream.ProcessExpiredPending("my-group", TimeSpan.FromMinutes(5));
```

`StreamManager` 的 `MaxRetryCount` 属性（默认 3）控制消息最大重试次数。

### Q8：KV 存储的过期数据如何清理？

KV 存储采用**惰性删除**策略：读取时发现过期则自动删除。也可主动调用清理：

```csharp
var cleaned = kv.CleanupExpired();
Console.WriteLine($"已清理 {cleaned} 个过期键");
```

建议定期（如每小时）调用 `CleanupExpired()` 回收磁盘空间。
