# NewLife.NovaDb 实现执行计划（从 需求规格说明书 落地到代码）

本文档基于 `Doc/需求规格说明书.md` 拆分出可执行、可迭代交付的实现计划。目标是让后续每一步都能形成可运行的增量（库可被引用、能创建库/表、能写入/查询、最终具备服务端/驱动/集群）。

---

## 里程碑（Milestones）

- **M0 仓库基线** ✅：解决方案结构清晰、统一编码风格、测试可运行。
- **M1 本地嵌入式最小可用（单表单索引）** ✅：文件夹即数据库 + 基础读写 + WAL + 恢复。
- **M2 事务/MVCC + 基础 SQL（DDL/DML/Select）** ✅：可并发读写、Read Committed。
- **M3 Nova Engine（SkipList）+ 索引冷热分离** ✅：热点内存索引 + 冷段 MMF 目录。
- **M4 自动切分/分片 + JOIN** ✅：单表逻辑无上限，分片路由、Nested Loop Join。
- **M5 Flux Engine（时序 + MQ 基础）** ✅：时间分片 Append Only + Stream/消费组/Pending/Ack。
- **M6 KV 视图 + TTL** ✅：KV API/SQL 视图、按行 TTL、惰性删除 + 后台清理。
- **M7 服务器模式 + ADO.NET** ✅：TCP RPC 协议 + ADO.NET Provider + 嵌入/服务器自动识别。
- **M8 集群与主从同步** ✅：WAL 流复制、一主多从、断点续传。
- **M9 SQL 函数体系** ⚠️：聚合/字符串/数值/日期/转换/条件/系统/地理向量/哈希函数。
- **M10 高级数据类型** ⚠️：GeoPoint 地理编码 + Vector 向量类型及索引。
- **M11 MQ 高级功能** ⚠️：延迟消息 + 死信队列 + 阻塞读取。
- **M12 KV 高级 API** ⚠️：Add（仅不存在时）+ Inc（原子递增） + 默认 TTL 策略。
- **M13 可观测性与管理工具** ⚠️：系统表查询 + Metrics + CLI 工具。

## 设计与实现原则

1. **先跑通再优化**：先实现正确的数据结构与持久化，再通过基准优化。
2. **模块边界清晰**：存储（页/编码/MMF/WAL）、事务（MVCC）、执行（SQL）、通信（协议/驱动）分层。
3. **纯托管**：不引入 native 依赖；跨平台优先（.NET 8+）。
4. **可测试**：每个模块都要有对应的单元测试与恢复测试（崩溃恢复/一致性）。
5. **多框架兼容**：主库 TFM 为 `net45;net461;netstandard2.0;netstandard2.1`，服务端/测试为 `net8.0`。

## 模块拆分（与需求 §2~§5 对应）

| 模块 | 目录 | 职责 |
|------|------|------|
| **Core** | `Core/` | 配置(`DbOptions`)、错误码(`ErrorCode`)、数据类型映射(`DataType`)、编解码(`IDataCodec`)、异常(`NovaException`) |
| **Storage** | `Storage/` | 文件布局(`DatabaseDirectory`/`TableDirectory`)、MMF(`MmfPager`)、页/块(`FileHeader`/`PageHeader`)、页缓存(`PageCache`) |
| **WAL** | `WAL/` | 日志写入(`WalWriter`)、记录格式(`WalRecord`)、崩溃恢复(`WalRecovery`) |
| **Tx** | `Tx/` | 事务管理(`TransactionManager`)、MVCC 行版本(`RowVersion`)、事务(`Transaction`) |
| **Engine.Nova** | `Engine/` | 表模型(`NovaTable`/`TableSchema`)、SkipList 索引、热/冷索引(`HotIndexManager`/`ColdIndexDirectory`)、分片(`ShardManager`/`ShardInfo`) |
| **Engine.Flux** | `Engine/Flux/` | 时序引擎(`FluxEngine`/`FluxEntry`)、消息ID(`MessageId`)、Stream/MQ(`StreamManager`/`ConsumerGroup`) |
| **Engine.KV** | `Engine/KV/` | KV 存储(`KvStore`/`KvEntry`) |
| **SQL** | `Sql/` | 词法(`SqlLexer`/`SqlToken`)、解析(`SqlParser`)、AST(`SqlAst`)、执行引擎(`SqlEngine`) |
| **Server** | `Server/` | TCP 服务(`NovaServer`/`NovaSession`)、RPC 控制器(`NovaController`)、协议定义(`NovaProtocol`) |
| **Client** | `Client/` | ADO.NET Provider(`NovaConnection`/`NovaCommand`/`NovaDataReader`/`NovaParameter`/`NovaTransaction`)、RPC 客户端(`NovaClient`) |
| **Cluster** | `Cluster/` | 主节点复制(`ReplicationManager`)、从节点客户端(`ReplicaClient`)、节点信息(`NodeInfo`) |

### 第九阶段：高级数据类型（M10）⚠️

#### 30) GeoPoint 地理编码类型 ⚠️

**目标**：对应需求 §4.1.2。

- DataType 枚举新增 `GeoPoint`（16 字节，双 Double）
- 编解码器支持 GeoPoint
- 计算函数：`DISTANCE()`、`WITHIN_RADIUS()`
- 索引：基于 GeoHash 或 R-Tree 的空间索引
- 适用场景：LBS 应用、附近查询、地理围栏

---

#### 31) Vector 向量类型 ⚠️

**目标**：对应需求 §4.1.3。

- DataType 枚举新增 `Vector`（定长 `Single[]`，如 `VECTOR(1536)` = 6144 字节）
- 编解码器支持 Vector
- 计算函数：`COSINE_SIMILARITY()`、`EUCLIDEAN_DISTANCE()`、`DOT_PRODUCT()`
- 索引：基于 HNSW 或 IVF 的向量索引，支持 KNN 查询
- 适用场景：AI 向量检索、语义搜索、推荐系统、RAG

---

### 第十阶段：MQ 高级功能（M11）⚠️

#### 32) 延迟消息 ⚠️

**目标**：对应需求 §3.2 延迟消息。

- 消息发送时指定 `DelayTime`（延迟秒数）或 `DeliverAt`（投递时间）
- 写入延迟队列（专用索引）
- 后台扫描线程按时间顺序轮询，到期消息自动转入正常队列
- 应用场景：订单超时取消、定时通知、重试退避

---

#### 33) 死信队列（DLQ）⚠️

**目标**：对应需求 §3.2 死信队列。

- 触发条件：消费失败达到最大重试次数（默认 3 次）或消费超时
- 存储结构：每个队列对应独立死信队列（`{QueueName}_DLQ`）
- 保留：原始消息 + 失败原因 + 失败次数
- 处理机制：独立订阅 + 重新投递（Retry）+ 永久删除
- 应用场景：异常排查、消息丢失防护、业务降级

---

#### 34) 阻塞读取 ⚠️

**目标**：对应需求 §3.2 阻塞读取。

- 支持阻塞 + 超时（类似 `XREADGROUP BLOCK`）
- 异步等待模型（`Task` / `CancellationToken`）
- 无新消息时挂起，有新消息或超时后返回

---

### 第十一阶段：KV 高级 API（M12）⚠️

#### 35) KV Add 与 Inc 操作 ⚠️

**目标**：对应需求 §3.3 KV 接口。

- `Add(key, value, ttl)`：仅当 key 不存在时添加，返回是否成功（分布式锁场景）
- `Inc(key, delta, ttl)`：原子递增（key 不存在时初始化为 delta，返回递增后的值）
- 默认 TTL 策略：未显式指定 TTL 时使用全局默认（如 24 小时），可在 `DbOptions` 配置

## 风险清单

| 风险 | 说明 | 缓解策略 |
|------|------|----------|
| **文件格式演进** | 一旦发布就要考虑兼容 | 文件头已有 Version 字段，需建立迁移策略 |
| **恢复与一致性** | WAL/Checkpoint 是正确性基石 | 已有崩溃恢复测试，需持续强化 |
| **冷热分离复杂度** | 目录结构与热段重建需精确定义 | 已有基础实现，需在大数据量下验证 |
| **SQL 子集边界** | 用户期望与实际支持的差距 | 文档明确 v1 支持范围，函数按优先级迭代 |
| **GeoPoint/Vector 索引** | 空间/向量索引算法复杂度高 | 先实现类型和计算函数，索引可分阶段 |
| **MQ 高级功能** | 延迟消息/死信队列增加系统复杂度 | 先完成基础 MQ，高级功能按需迭代 |
