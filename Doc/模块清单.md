# NewLife.NovaDb 模块清单

> 基于《需求规格说明书.md》，以"可对外验收能力"为中心进行模块化拆分。
> 领域+模块两级编码，便于后续开发跟踪。

---

## 1. 编码规则

模块 ID = 领域前缀 + 两位序号（如 `C01`、`S03`）。

| 前缀 | 领域 | 代码目录 |
|------|------|---------|
| **C** | Core — 类型系统/错误/配置 | `Core/` |
| **S** | Storage — 文件布局/页读写/缓存/版本 | `Storage/` |
| **L** | Log — WAL/Binlog/崩溃恢复 | `WAL/` |
| **T** | Tx — 事务/MVCC/只读/跨进程锁 | `Tx/` |
| **N** | Nova — 关系型引擎/表/索引/CRUD | `Engine/` |
| **F** | Flux — 时序+MQ/分片/消费组 | `Engine/Flux/` |
| **K** | KV — 键值+TTL | `Engine/KV/` |
| **Q** | SQL — 词法/语法/执行器/函数 | `Sql/` |
| **P** | Provider — 服务器/协议/ADO.NET | `Server/` + `Client/` |
| **R** | Replica — 主从复制/集群 | `Cluster/` |
| **O** | Ops — 运维/可观测性 | 分布在各层 |
| **X** | eXtra — 物化视图/Geo/Vector/安全 | 分布在各层 |

## 2. 模块清单

### 2.1 Core（基础设施）

| ID | 模块 | 优先级 | 依赖 | 验收口径 |
|----|------|--------|------|---------|
| C01 | 类型系统与值表示 | P0 | — | 支持 BOOL/INT/LONG/DOUBLE/DECIMAL/STRING/BINARY/BLOB/DATETIME，含 NULL 语义与 C# 映射 |
| C02 | 编解码与类型转换 | P0 | C01 | 值的二进制编解码稳定；SQL CAST/隐式转换规则一致 |
| C03 | 异常与错误码 | P0 | — | 语法/对象不存在/只读/约束冲突/超时/事务冲突等可区分，错误码稳定 |
| C04 | 配置模型（DbOptions） | P0 | — | WAL 等级、缓存上限、慢查询阈值等可配置，默认值可用 |
| C05 | 时间与基础服务 | P0 | — | NOW()/随机等非确定性函数依赖统一源 |

### 2.2 Storage（存储与文件）

| ID | 模块 | 优先级 | 依赖 | 验收口径 |
|----|------|--------|------|---------|
| S01 | 文件夹即数据库 | P0 | — | 数据库目录可拷贝迁移；文件按约定落盘 |
| S02 | 表文件组管理 | P0 | S01 | 每表独立文件组（.data/.idx/.wal），表间隔离 |
| S03 | 页读写与分页访问 | P0 | S01 | MMF 按页访问超大文件；PageIO 抽象稳定 |
| S04 | 页缓存与淘汰 | P0 | S03, C04 | LRU 缓存可配置；压力下不 OOM；暴露命中率 |
| S05 | 元数据持久化 | P0 | S01 | 持久化库/表/索引目录；支持系统表查询 |
| S06 | 数据文件版本兼容 | P0 | S01 | 文件头含版本号；新版打开旧版自动升级；不兼容明确拒绝 |

### 2.3 Log（WAL/Binlog）

| ID | 模块 | 优先级 | 依赖 | 验收口径 |
|----|------|--------|------|---------|
| L01 | WAL 写入 | P0 | S03, C04 | FULL/NORMAL/NONE 三种持久性可配置 |
| L02 | 崩溃恢复 | P0 | L01, T01 | 重启后已提交可见、未提交不可见 |
| L03 | Checkpoint/刷盘 | P0 | L01 | 不破坏一致性；控制 WAL 增长；异常可重入 |
| L04 | Binlog 记录与滚动 | P0 | S01, T01 | 可滚动/清理；用于复制/增量恢复/审计 |
| L05 | Binlog 管理命令 | P0 | L04, Q03 | SHOW BINLOGS/STATUS、PURGE、FLUSH；清理幂等 |

### 2.4 Tx（事务与一致性）

| ID | 模块 | 优先级 | 依赖 | 验收口径 |
|----|------|--------|------|---------|
| T01 | 事务管理器 | P0 | C03, S05, L01 | BEGIN/COMMIT/ROLLBACK；未显式开启自动提交 |
| T02 | MVCC 行版本与可见性 | P0 | T01 | Read Committed：不读脏/语句级一致/读己之写；写写冲突可重试 |
| T03 | 嵌入跨进程单写协调 | P0 | S02 | 表级文件锁；只读不阻塞读 |
| T04 | 只读连接语义 | P0 | T01 | 拒绝 DDL/DML 返回明确错误；允许查询 |
| T05 | DDL 事务约定 | P0 | Q03, T01 | DDL 隐式提交/不可回滚 |

### 2.5 Nova（关系型引擎）

| ID | 模块 | 优先级 | 依赖 | 验收口径 |
|----|------|--------|------|---------|
| N01 | 表模型与行格式 | P0 | S05, C01 | CREATE TABLE 元数据；列类型/NULL/DEFAULT/AUTO_INCREMENT/COMMENT |
| N02 | 主键索引与点查 | P0 | N01, S03 | SkipList 主键索引；点查 P95 < 5ms（百万行热数据） |
| N03 | 基础 CRUD | P0 | N01, T01, L01 | INSERT/UPDATE/DELETE/UPSERT/TRUNCATE；影响行数正确 |
| N04 | 二级索引与唯一索引 | P1 | N01, N03 | 普通/唯一/联合唯一索引；查询可使用索引 |
| N05 | 索引冷热分离 | P0 | N02, S04 | 热点缓存加速，冷数据逐出；内存可控可配置 |
| N06 | 分片管理 | P0 | N01, S02 | 单表逻辑无上限；按大小/行数自动切分 |

### 2.6 Flux（时序 + MQ）

| ID | 模块 | 优先级 | 依赖 | 验收口径 |
|----|------|--------|------|---------|
| F01 | 时序数据模型 | P0 | S03, T01 | 追加写入 + 时间范围查询；按时间组织 |
| F02 | 时间分片管理 | P0 | F01 | 按天/小时分片；分片可控 |
| F03 | 保留策略与 TTL | P0 | F02 | 按时间/容量自动清理；过期对读不可见 |
| F04 | Stream/MQ 核心 | P1 | S05, T01 | 消费组/Pending/Ack；At-Least-Once；严格顺序（融合到 NovaQueue） |
| F05 | 阻塞读取与超时 | P1 | F04 | 阻塞+超时；连接可复用（融合到 NovaQueue） |
| F06 | 延迟消息与 DLQ | P1 | F04 | 延迟投递、死信队列、重试机制（融合到 NovaQueue） |
| F07 | 降采样 | P1 | F02 | 历史数据降采样，降低存储与查询成本 |
| F08 | 时间窗口聚合 | P1 | F01, Q07 | 滚动窗口、滑动窗口聚合 |

### 2.7 KV（键值）

| ID | 模块 | 优先级 | 依赖 | 验收口径 |
|----|------|--------|------|---------|
| K01 | KV 表与 TTL | P1 | S05, T01 | Bitcask 模型持久化：内存索引+磁盘值+CRC32+回放+过期跳过+自动压缩；多表隔离 |
| K02 | KV API | P1 | K01 | Set/Get/Delete/Exists/Add/Replace/Inc/IncDouble/Expiration/批量/搜索；API 与 SQL 行为一致 |

### 2.8 SQL（解析与执行）

| ID | 模块 | 优先级 | 依赖 | 验收口径 |
|----|------|--------|------|---------|
| Q01 | SQL 方言约定 | P0 | — | 大小写不敏感；双引号标识符；`@name` 参数；ENGINE 选择 |
| Q02 | 词法/语法/AST | P0 | Q01 | 解析 P0 DDL/DML/SELECT；语法错误可区分 |
| Q03 | DDL 执行器 | P0 | Q02, S05, T05 | CREATE/DROP DB/TABLE、ALTER TABLE、CREATE/DROP INDEX |
| Q04 | DML 执行器 | P0 | Q02, N03, T01 | INSERT/UPDATE/DELETE/UPSERT/TRUNCATE |
| Q05 | SELECT 执行器 | P0 | Q02, N02 | WHERE/ORDER BY/GROUP BY/HAVING/LIMIT/OFFSET；参数化 |
| Q06 | 表达式与类型推断 | P0 | C01, C02 | 表达式计算与 NULL 传播；类型转换规则一致 |
| Q07 | SQL 函数系统 | P0 | Q06 | 函数注册+执行；覆盖 P0 函数集合（见 SQL 函数清单） |
| Q08 | JOIN 与子查询 | P1 | Q05 | INNER/LEFT/RIGHT JOIN；IN/EXISTS 子查询 |
