# NewLife.NovaDb 模块清单

> 本文档基于《需求规格说明书.md》，以"可对外验收能力"为中心进行模块化拆分。
>
> - 不写实现细节：页格式/索引结构/协议字段等细节留给《架构设计文档.md》及 `Doc/架构/` 目录下独立文档。
> - 模块 ID 仅用于标识与引用，不代表实施顺序。
>
> 更新日期：2026年2月23日

---

## 1. 模块编码规则

模块 ID 由"前缀 + 两位序号"组成：

| 前缀 | 领域 |
|---|---|
| **C** | Core：通用基础设施（类型系统/错误/配置等） |
| **S** | Storage：文件夹即数据库、文件组、页读写、缓存、版本兼容 |
| **L** | Log：WAL/Binlog、崩溃恢复、清理与滚动 |
| **T** | Tx：事务/MVCC/只读语义/嵌入跨进程写协调 |
| **N** | Nova：通用关系型引擎（表/索引/CRUD/冷热分离） |
| **F** | Flux：时序 + MQ（分片/TTL/消费组/降采样/时间窗口） |
| **K** | KV：Key/Value + TTL（SQL + API 访问） |
| **Q** | SQL：词法/语法/表达式/执行器/函数系统 |
| **P** | Provider：服务器进程 + 协议 + ADO.NET Provider |
| **R** | Replica：主从复制/集群/故障切换 |
| **O** | Ops：运维与可观测性（EXPLAIN/慢查询/系统表/指标/备份） |
| **X** | eXtra：物化视图、Geo/Vector、安全等 |

字段说明：

- **依赖**：最小依赖集（用于拓扑排序）。
- **验收口径**：外部可观测/可测试的行为，不写内部实现方式。

## 2. 模块清单（按领域分组）

### 2.1 Core（基础设施）

| ID | 模块 | 优先级 | 需求来源 | 依赖 | 验收口径（摘要） |
|---|---|---|---|---|---|
| **C01** | 类型系统与值表示 | P0 | §4.1 | — | 支持核心类型（BOOL/INT/LONG/DOUBLE/DECIMAL/STRING/BINARY/BLOB/DATETIME），含 NULL 语义与 C# 映射一致性 |
| **C02** | 编解码与类型转换基础 | P0 | §4.1, §5 | C01 | 存储层/执行器可对值进行稳定编解码；SQL CAST/隐式转换具备一致规则（细则见函数清单） |
| **C03** | 异常与错误码体系 | P0 | §5.12 | — | 对外错误可区分（语法/对象不存在/只读拒绝/约束冲突/超时/事务冲突等），错误码与消息结构稳定 |
| **C04** | 配置模型（DbOptions） | P0 | §2, §6 | — | 可配置 WAL 等级、缓存上限、慢查询阈值等关键参数；默认值可用且有边界校验 |
| **C05** | 时间与随机等基础服务 | P0 | §5.10, §6 | — | NOW()/随机等非确定性函数依赖统一时间源/随机源，便于测试与可观测 |

| **S04** | 页缓存与淘汰策略 | P0 | §2, §6.3 | S03, C04 | 缓存命中/淘汰可配置；压力下保持稳定（不 OOM），并向指标系统暴露命中率等 |
| **S05** | 元数据持久化（库/表/索引目录） | P0 | §5.1, §5.10 | S01 | 支持持久化数据库对象目录；支持系统表/SHOW 查询元数据（详见 O03） |

### 2.3 Log（WAL/Binlog）

| ID | 模块 | 优先级 | 需求来源 | 依赖 | 验收口径（摘要） |
|---|---|---|---|---|---|
| **L01** | WAL 写入（FULL/NORMAL/NONE） | P0 | §2, §3.4 | S03, C04 | 三种持久性等级可配置；FULL 同步语义明确；崩溃后可恢复到一致状态（配合 L02） |
| **L02** | 崩溃恢复 | P0 | §5.3 | L01, T01 | 重启后：已提交事务可见，未提交不可见；恢复过程可诊断（日志/事件） |
| **L03** | Checkpoint/刷盘点 | P0 | §2 | L01 | Checkpoint 不破坏一致性；可控制 WAL 增长；异常中断可重入 |
| **L04** | Binlog 记录与滚动 | P0 | §2, §5.5 | S01, T01 | Binlog 可滚动/可清理；用于复制/增量恢复/审计追踪；格式版本化（兼容策略见 §7） |
| **L05** | Binlog 管理命令 | P0 | §5.5 | L04, Q03 | 支持 SHOW BINLOGS/STATUS、PURGE BINLOGS TO/BEFORE、FLUSH BINLOGS；清理幂等；越界返回明确错误并返回清理范围/数量 |

### 2.4 Tx（事务与一致性）

| ID | 模块 | 优先级 | 需求来源 | 依赖 | 验收口径（摘要） |
|---|---|---|---|---|---|
| **T01** | 事务管理器 | P0 | §5.3 | C03, S05, L01 | 支持 BEGIN/COMMIT/ROLLBACK；未显式开启事务时自动提交；事务生命周期与错误语义明确 |
| **T02** | MVCC 行版本与可见性 | P0 | §2, §5.3 | T01 | Read Committed：不读脏数据、语句级一致性、读己之写；写写冲突返回明确可重试错误；读写尽量不互相阻塞 |
| **T03** | 嵌入模式跨进程单写协调 | P0 | §2 | S02 | 嵌入多进程场景：写入需有表级文件锁/等价机制；只读连接不阻塞读 |
| **T04** | 只读连接语义 | P0 | §5.4 | T01 | 只读连接拒绝所有 DDL/DML 并返回明确错误；允许查询与元数据读取 |
| **T05** | DDL 事务语义约定 | P0 | §5.3 | Q03, T01 | v1 默认 DDL 为隐式提交/不可回滚；行为一致并在错误中可诊断 |

### 2.5 Nova（关系型引擎）

| ID | 模块 | 优先级 | 需求来源 | 依赖 | 验收口径（摘要） |
|---|---|---|---|---|---|
| **N01** | 表模型与行格式 | P0 | §3.1, §4.3, §5.1 | S05, C01 | 支持 CREATE TABLE 元数据映射；列类型/NULL/DEFAULT/AUTO_INCREMENT/COMMENT 语义一致 |
| **N02** | 主键索引与点查 | P0 | §3.1, §6.1 | N01, S03 | 支持主键点查性能目标（P95 < 5ms）；主键唯一性约束可诊断 |
| **N03** | 基础 CRUD（Insert/Update/Delete） | P0 | §5.1, §6.1 | N01, T01, L01 | DML 正确影响行数；支持 UPSERT/TRUNCATE；与事务/只读/约束冲突的错误语义一致 |
| **N04** | 二级索引与唯一索引 | P1 | §3.1, §4.3 | N01, N03 | 支持普通索引与唯一索引（含联合唯一索引）；查询可使用索引（与 EXPLAIN 联动） |
| **N05** | 索引冷热分离 | P0 | §2, §3.1 | N02, S04 | 频繁访问的热点索引通过缓存加速，冷数据主动逐出内存；内存占用可控可配置 |

> **说明**：需求规格书明确"不支持 Nova Engine 自动水平分片/分库分表"（§5.11），大规模场景由应用层分表或使用 Flux 分段能力实现。

### 2.6 Flux（时序 + MQ）

| ID | 模块 | 优先级 | 需求来源 | 依赖 | 验收口径（摘要） |
|---|---|---|---|---|---|
| **F01** | 时序数据模型（追加写） | P0 | §3.2.1 | S03, T01 | 支持追加写与时间范围查询；数据按时间组织（对外语义明确） |
| **F02** | 时间分片管理 | P0 | §3.2.1 | F01 | 支持按天/小时等策略组织分片；分片可控（配置/默认策略） |
| **F03** | 保留策略与 TTL 清理 | P0 | §3.2.1 | F02 | 支持按时间/容量自动清理；过期数据对读取不可见（允许惰性/后台清理） |
| **F04** | Stream/MQ 核心语义 | P1 | §3.2.2, §5.6 | S05, T01 | 支持消费组/消费者/Pending/Ack；At-Least-Once；严格顺序投递（单表不分区）；位点自动管理 |
| **F05** | 阻塞读取与超时 | P1 | §3.2.2, §5.6 | F04 | 支持阻塞读取/超时语义；不会导致连接不可复用 |
| **F06** | 延迟消息与 DLQ | P1 | §3.2.2, §5.6 | F04 | 支持延迟投递、死信队列（DLQ）、重试机制 |
| **F07** | 降采样（Down-Sampling） | P1 | §3.2.1 | F02 | 支持对历史数据进行降采样，降低存储与查询成本 |
| **F08** | 时间窗口聚合 | P1 | §3.2.1 | F01, Q07 | 支持滚动窗口、滑动窗口等时间窗口聚合查询 |

> **说明**：连续查询（Continuous Query，§3.2.1）的能力由 X01 增量物化视图覆盖。MQ 不承诺 Exactly-Once 端到端语义（§5.6），由业务侧幂等实现。

### 2.8 SQL（方言、解析与执行）

| ID | 模块 | 优先级 | 需求来源 | 依赖 | 验收口径（摘要） |
|---|---|---|---|---|---|
| **Q01** | SQL 方言约定 | P0 | §5.1 | — | 关键字不区分大小写；标识符大小写不敏感；双引号引用标识符（兼容 SQL-92）；`--` 注释；命名参数 `@name`；ENGINE=NOVA\|FLUX\|KV |
| **Q02** | 词法/语法/AST | P0 | §5.1 | Q01 | 可解析 P0 DDL/DML/SELECT 子集；语法错误返回可区分的错误类型（C03） |
| **Q03** | DDL 执行器 | P0 | §5.1 | Q02, S05, T05 | 支持 CREATE/DROP DATABASE/TABLE、ALTER TABLE（ADD/MODIFY/DROP COLUMN、COMMENT）、CREATE/DROP INDEX（含 UNIQUE INDEX）；DDL 隐式提交语义一致 |
| **Q04** | DML 执行器 | P0 | §5.1 | Q02, N03, T01 | 支持 INSERT/UPDATE/DELETE/UPSERT/TRUNCATE；影响行数可观测 |
| **Q05** | SELECT 执行器（P0 子集） | P0 | §5.1 | Q02, N02, N03 | 支持 WHERE/ORDER BY/GROUP BY/HAVING/LIMIT/OFFSET；参数化查询 |
| **Q06** | 表达式与类型推断/隐式转换 | P0 | §4, §5 | C01, C02, Q05 | 表达式计算与 NULL 传播一致；类型不匹配错误/返回 NULL 规则按函数清单约定 |
| **Q07** | SQL 函数系统（注册+执行） | P0 | §4.2, §5.12 | Q06 | 内置函数可按名称注册；覆盖 P0 函数集合（字符串/数值/日期时间/聚合/条件/类型转换/系统元数据）；错误/NULL 规则一致；具体清单见《SQL函数支持清单.md》 |
| **Q08** | JOIN 与子查询 | P1 | §5.2 | Q05 | 支持 INNER/LEFT/RIGHT JOIN 与多表连续 JOIN；支持 IN/EXISTS 子查询；能力边界明确 |

### 2.10 Replica（集群与主从）

|---|---|---|---|---|---|

### 2.11 Ops（运维与可观测性）

| ID | 模块 | 优先级 | 需求来源 | 依赖 | 验收口径（摘要） |
|---|---|---|---|---|---|
| **O01** | EXPLAIN / EXPLAIN ANALYZE | P0 | §5.10 | Q05 | 输出至少包含：访问对象、访问方式（全表/索引）、预估行数、执行顺序；格式稳定（可追加字段） |
| **O02** | 慢查询日志 | P0 | §5.10 | C04, Q05 | 支持阈值配置；记录 SQL、执行时间、扫描行数/返回行数（可用则记）、命中索引信息（可选） |
| **O03** | 系统表/SHOW 元数据 | P1 | §5.10 | S05, Q05 | 可查询表列表/索引列表/连接列表/事务列表/WAL-Binlog 状态/数据文件统计等元信息；Schema 稳定可演进 |
| **O04** | 监控指标（Metrics） | P1 | §5.10 | C05 | 性能指标（QPS/延迟分位）、IO 吞吐、网络吞吐（服务器模式）、资源占用、业务指标（表行数/索引大小/慢查询计数/事务冲突/MQ 堆积/物化视图延迟等） |
| **O05** | 备份与恢复 | P1 | §5.10 | S01, L04 | 离线备份 = 拷贝目录；增量恢复基于 Binlog |

### 2.12 eXtra（物化视图、Geo/Vector、安全）

| ID | 模块 | 优先级 | 需求来源 | 依赖 | 验收口径（摘要） |
|---|---|---|---|---|---|
| **X01** | 增量物化视图 | P1 | §3.2.3, §5.8 | F01, Q05, Q07 | 支持 CREATE/REFRESH/DROP MATERIALIZED VIEW；增量捕获源表变更（INSERT/UPDATE）并更新统计表；支持增量聚合函数（SUM/COUNT/AVG/MIN/MAX）；支持多维度 GROUP BY（时间维度 + 业务维度，含 time_bucket）；同时覆盖连续查询（Continuous Query）能力 |
| **X02** | 物化视图定时修正与多粒度汇总 | P1 | §3.2.3, §5.8 | X01 | 支持修正调度配置（cron 表达式）与修正时间窗口；修正与增量更新互斥（锁机制）；支持多层次时间粒度聚合（分钟→小时→天）；支持配置聚合缓冲区大小与刷新间隔；可通过系统表（`_sys.materialized_views`）查询视图状态 |
| **X03** | GeoPoint 类型与函数 | P2 | §4.1.2 | C01, Q07 | 支持 GEOPOINT 类型（经纬度坐标）与距离/范围/多边形/最近邻查询函数；支持地理索引；性能目标：百万级范围查询 P95 < 100ms |
| **X04** | Vector 类型与函数 | P2 | §4.1.3 | C01, Q07 | 支持 VECTOR(n) 类型（1~4096 维浮点数组）与相似度函数（L2/余弦/内积）；支持 KNN Top-K 查询；性能目标：百万级 KNN(K=10) P95 < 500ms |
| **X05** | 认证与鉴权 | P2 | §5.13 | P01 | 服务器模式支持认证（账号/Token）与权限控制（库/表粒度），不进入 v1 验收 |
| **X06** | TLS 传输加密 | P2 | §5.13 | P01 | TLS 可选扩展，不进入 v1 验收 |
